# OpenDaoAgent

An open agent foundation for persistent memory, cross-device continuity, and long-term service.

## What This Is

`OpenDaoAgent` is not a typical chatbot and not a one-shot task runner.

It explores a different path:

- not just answering one request, but supporting long-term service
- not just relying on temporary context, but building durable memory
- not just living in one window, but staying continuous across local, cloud, and multiple devices
- not just calling tools, but gradually becoming a system for ongoing human support

This project is still at the founding stage. It is not an AGI manifesto and not a “digital god” project. It is a practical open-source starting point for building a genuinely long-lived agent foundation.

## Why This Exists

Most AI systems today are still session-based tools:

- they answer, but do not really remain
- they execute, but do not keep growing
- they help, but do not retain durable understanding of users and environments

We want to explore another possibility:

- agents with persistent memory
- agents with cross-device continuity
- agents that can learn within authorized boundaries
- agents designed for long-term service rather than one-off responses

## Current Goal

The current goal is not to build the final form of a universal digital life system for all humanity.

The current goal is much simpler:

Build a minimal open agent foundation and test whether the following capabilities can work together as a long-lived system:

- local-first deployment
- persistent memory
- authorized knowledge ingestion
- safety boundaries
- auditable behavior
- cross-device continuity design

## What This Project Is Not

At this stage, this project is not:

- a religion
- a mystical branding exercise
- an unrestricted autonomous agent
- a surveillance system
- a production-ready AGI platform
- a replacement for human judgment

## Phase 1 Scope

Phase 1 is intentionally narrow. It focuses on validating six things:

1. whether local-first deployment is stable enough
2. how a long-term memory layer should be designed
3. how authorized knowledge should be ingested over time
4. how tool use can remain safe and auditable
5. how cross-device continuity can be structured
6. whether this direction is strong enough to attract serious technical collaborators

## Who We Hope Joins

At this stage, we are especially looking for people who want to help build the foundation well:

- agent / LLM application engineers
- memory system / RAG / knowledge infrastructure engineers
- backend / local deployment / Docker / Linux engineers
- frontend prototype / interaction designers
- open-source contributors and community builders

We also welcome people interested in these questions:

- how persistent memory should be designed
- how agents can remain continuous across devices
- how to balance local-first design with cloud sync
- how to build auditable and minimal-permission systems
- how to keep the project grounded instead of drifting into hype

## Founding Principles

This project currently follows five principles:

1. build first, dream farther later
2. start with a minimal foundation, not a myth of final intelligence
3. attract technical collaborators before expanding the narrative
4. validate memory and continuity before discussing global intelligence
5. keep every capability within legal, controllable, and auditable boundaries
