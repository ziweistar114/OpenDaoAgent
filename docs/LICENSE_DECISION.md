# 许可证选择建议

## 当前建议

如果你准备把 `OpenDaoAgent` 公开到 GitHub，建议在首发前就明确许可证。

否则别人虽然能看到代码，但不一定敢参与贡献或复用。

## 常见选择

### 1. MIT

优点：

- 最宽松
- 最容易让别人快速参与
- 对早期开源项目最省事

缺点：

- 保护性弱
- 别人更容易拿去做闭源商业包装

### 2. Apache-2.0

优点：

- 相对宽松
- 比 MIT 更正式
- 对专利相关条款更清楚

缺点：

- 比 MIT 稍重一点

### 3. GPL

优点：

- 强制衍生项目继续开源

缺点：

- 对商业合作和生态扩展限制更强
- 早期可能降低参与门槛

## 对你当前阶段的建议

如果你的目标是：

- 先把项目发出来
- 先吸引技术同路人
- 先做最小原型

那么更建议优先考虑：

- `MIT`
或
- `Apache-2.0`

## 我的倾向

如果你想要“先开放、先聚人、先把生态长出来”，我更建议：

### 第一选择：Apache-2.0

原因：

- 比 MIT 更稳一点
- 对技术项目首发更体面
- 仍然足够开放

### 第二选择：MIT

原因：

- 最省事
- 最容易快速开始

## 当前处理建议

在你没有最终想清楚之前，可以先：

1. 用本文件记录选择建议
2. 首发前确认一次
3. 再补正式 `LICENSE` 文件

如果你想更稳妥，建议在正式公开前再确认一次许可证，不要随手写死。
